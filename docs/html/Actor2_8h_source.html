26 /*
27  This form of _declspec can be applied to any class declaration, but should
28  only be applied to pure interface classes, that is classes that will never
29  be instantiated on their own. The _declspec stops the compiler from
30  generating code to initialize the vfptr in the constructor(s) and
31  destructor of the class. In many cases, this removes the only references
32  to the vtable that are associated with the class and, thus, the linker will
33  remove it. Using this form of _declspec can result in a significant
34  reduction in code size.
